# Sail Recall

Поиск похожих гонок по условиям и синтез тренерских советов из накопленного опыта.
Это не список файлов — связный брифинг.

## Конфигурация

```bash
SAILING_REPO="/Users/vita/projects/sailgpx"
source "$SAILING_REPO/tooling/sailenv.sh"   # → SAILING_DIR, SAILING_TRACKS_DIR
```

Источник: `$SAILING_DIR/.race-index.json` (словарь `races`). Заметки гонок —
`$SAILING_TRACKS_DIR/<slug>/<slug>.md`.

## Workflow

1. Сформируй **query**:
   - если вызвано из/для конкретной гонки — возьми её `venue`, `wind_dir_card`,
     `wind_bucket` (из `weather_forecast` или `weather_actual`), `course_type`,
     `distance_nm`, `class`;
   - либо из аргументов пользователя (напр. «W 10–15 на Минском море, ВЛ-дистанция»).
2. Ранжируй похожие:

```bash
python3 - <<PY
import sys, json, os; sys.path.insert(0, "$SAILING_REPO/lib")
from sailrecall.core import find_similar
idx_path = os.path.join("$SAILING_DIR", ".race-index.json")
idx = json.load(open(idx_path)) if os.path.exists(idx_path) else {}
query = {"venue": "<venue>", "wind_dir_card": "<card>", "wind_bucket": "<bucket>",
         "course_type": "<course_type>", "distance_nm": <dist>, "class": "<class>"}
for r in find_similar(idx, query, exclude="<current_slug_or_None>"):
    print(json.dumps(r, ensure_ascii=False))
PY
```

3. Для топ-N кандидатов **открой заметки** (`<slug>/<slug>.md`) и извлеки:
   - секцию `## Мысли и инсайты` (свободный текст пользователя),
   - `result` (место/флот),
   - блок «Анализ трека» (углы лавировки, groove, что теряли).
4. **Синтезируй брифинг** (не перечисление файлов):
   - повторяющиеся инсайты при похожем ветре/акватории;
   - что работало (выгодный галс/сторона, тактика старта, огибания);
   - типичные ошибки в этих условиях;
   - 2–4 конкретных совета на новую гонку.
   Сослаться на гонки-источники как `[[<slug>]]`.
5. Если похожих мало (`< 2`) — честно сказать, что данных пока недостаточно, и какие
   условия записать в будущем.

## Правила

- Не выдумывать инсайты — только из заметок. Нет данных → сказать прямо.
- Только чтение; ничего в заметках не менять (кроме случая, когда `/sail-race new`
  просит вставить брифинг в раздел «Прогноз» текущей гонки — тогда писать в auto-секцию).

## Обработка ошибок

| Ошибка | Решение |
|--------|---------|
| Пустой/нет `.race-index.json` | Сообщить, что история пуста |
| Нет подходящих (score<min) | Снизить требования (только venue+ветер) или сказать «похожих нет» |
| Заметка-кандидат отсутствует | Пропустить, отметить рассинхрон индекса |
